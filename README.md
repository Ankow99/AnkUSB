# AnkUSB

A Ventoy‑based USB toolkit preloaded with a handpicked selection of Linux distribution ISOs—spanning desktop, enterprise, handheld/HTPC, storage appliance, security/live‑forensics, and utility systems—plus Tiny Windows installer images, all ready for instant boot and deployment on any x86_64 machine. 

> Theme based on based on [CRT‑Amber‑GRUB‑Theme](https://github.com/Jacksaur/CRT-Amber-GRUB-Theme), special thanks to its creator [Jackson](https://github.com/Jacksaur)!

# Installation

### Prerequisites

- A USB drive of at least 8 GiB — empty or pre‑formatted. (To include the same ISOs as this repo, a 256 GiB USB is required)  
- A host machine with **Windows** (any recent edition) or **Linux** (any modern distribution) and administrator/root privileges.  
- Internet access to download:  
  - The latest [Ventoy installer](https://www.ventoy.net/en/download.html).  
  - Your desired ISO images (e.g., Arch Linux; Fedora Workstation—see table above).  
- A clone of the AnkUSB repository (contains only the custom Ventoy theme and modified JSON).  

---

## 1. Install Ventoy

### Install Ventoy on Windows (GUI)

1. **Download** the Windows ZIP from the [Ventoy website](https://www.ventoy.net/en/download.html).
   
2. **Unzip** the archive and run `Ventoy2Disk.exe` (double‑click).
   
   - This opens a graphical installer  
   - In the **Device** dropdown, select your USB drive  
   - Click **Install** to deploy Ventoy onto the USB stick
     
3. **Confirm** when prompted that all data on the USB will be overwritten.

   > **Tip:** I also suggest preserving some space at the bottom of the disk for general use. For my 256 GiB USB, I left a 50 GiB exFAT partition.  

   > **What this does:** the GUI installs Ventoy’s bootloader onto your USB, creating two partitions (one EFI/bootable and one for your ISOs).  


### Install Ventoy on Linux (CLI)

1. **Download** the Linux tarball from the [Ventoy website](https://www.ventoy.net/en/download.html).  
2. **Identify** your USB device node:
   
   ```sh
   $ lsblk
   ```
   
3. **Extract** the Ventoy installer:
   
   ```sh
   $ tar -xzf ventoy-*.tar.gz
   ```
   - tar decompresses the archive, creating a ventoy-* directory.
  
4. **Install** Ventoy onto the USB (`sdX` is the USB device, for example `/dev/sdb`):
   
   ```sh
   # ./Ventoy2Disk.sh -i -sg /dev/sdX
   ```
   ```
   Ventoy2Disk.sh CMD [ OPTION ] /dev/sdX
     CMD:
       -i   install ventoy to sdX (fail if disk already installed with ventoy)
       -I   force install ventoy to sdX (no matter installed or not)
       -u   update ventoy in sdX
       -l   list Ventoy information in sdX
      
     OPTION: (optional)
      -r SIZE_MB  preserve some space at the bottom of the disk (only for install)
      -s          enable secure boot support (default is disabled)
      -g          use GPT partition style, default is MBR style (only for install)
      -L          Label of the main partition (default is Ventoy)
   ```
   > **Tip:** I also suggest preserving some space at the bottom of the disk for general use. For my 256 GiB USB, I left a 50 GiB exFAT partition.  

   > **What this does:** the script installs Ventoy’s bootloader onto your USB, creating two partitions (one EFI/bootable and one for your ISOs).  


### Install Ventoy on Linux (GUI)

1. **Download** the Linux tarball from the [Ventoy website](https://www.ventoy.net/en/download.html).  
2. **Extract** the Ventoy installer:
   
   ```sh
   $ tar -xzf ventoy-*.tar.gz
   ```
   - tar decompresses the archive, creating a ventoy-* directory.
    
3. **Execute** `VentoyGUI.x86_64`:
   
   ```sh
   # ./VentoyGUI.x86_64
   ```
   
   - This opens a graphical installer  
   - In the **Device** dropdown, select your USB drive  
   - Click **Install** to deploy Ventoy onto the USB stick
     
4. **Confirm** when prompted that all data on the USB will be overwritten.

   > **Tip:** I also suggest preserving some space at the bottom of the disk for general use. For my 256 GiB USB, I left a 50 GiB exFAT partition.  

   > **What this does:** the GUI installs Ventoy’s bootloader onto your USB, creating two partitions (one EFI/bootable and one for your ISOs).

## 2. Clone the AnkUSB Repository

```sh
$ git clone https://github.com/Ankow99/AnkUSB.git
```
This repository includes only:

1. A custom Ventoy theme (`/ventoy/theme/CRT-Amber/`).
2. A custom Ventoy config (`/ventoy/ventoy.json`).
3. Two bootable images as a workaround (2) for legacy BIOSes with 137 GB limitation (`/ventoy/ventoy.disk.img.xz`, `/ventoy/ventoy_4k.disk.img.xz`). See [legacy BIOS limitation](https://www.ventoy.net/en/doc_legacy_limit.html).
4. A patched `unicode.pf2` font meant to replace `VTOYEFI/grub/fonts/unicode.pf2`.
5. A modified JSON template easier to edit (`/ventoy_template.json`). (Optional)
6. A ISO list with the ISOs I used (`/ISOs-List.txt`). (Optional)

You will later merge these into your Ventoy USB layout.

---

## 3. Download Your ISO Images

For space and obvious reasons, the AnkUSB repo does not include ISOs. Instead, manually download the latest images from their official repo.

Place each ISO in its respective directory as indicated in the `\ventoy.json` file. By default, I used the following:

- Desktop Linux: `/ISOs/Desktop/`
- Enterprise Linux: `/ISOs/Enterprise/`
- Handheld Linux: `/ISOs/Handheld/`
- Pentesting Linux: `/ISOs/Security/`
- Storage Appliances: `/ISOs/Storage/`
- Utilities: `/ISOs/Utilities/`
- Windows: `/ISOs/Windows/`

---

## 4. Enable Persistence (Optional)

To retain changes in supported live‑session distributions:

1. Download the Ventoy persistence data files from the [official backend releases](https://github.com/ventoy/backend/releases).

2. Place persistence files under `/Persistence` on the USB stick.

3. Edit `/ventoy/ventoy.json` to include persistence entries.

---

## 5. Patching the Default Ventoy Font

Simply replace the file `\VTOYEFI/grub/fonts/unicode.pf2` (on the second EFI/bootable partition) on your Ventoy USB with `unicode.pf2` from this repository.

---

## 6. Copy Files to Your Ventoy USB

1. Mount the Ventoy USB:
   
   - Windows: appears as a new drive (e.g. E:\)
   - Linux: auto‑mounted under `/media/$USER/Ventoy` or mount manually
  
2. Copy the repository contents into the USB root, preserving directory structure.
   
   Ensure the following paths on the USB:

   - `/ISOs/*` — Your downloaded ISO files.
   - `/ventoy/ventoy.json` — Ventoy's custom config.
   - `/ventoy/theme/CRT-Amber/theme.txt` — Custom CRT theme.
   - `/ventoy/ventoy.disk.img.xz` & `/ventoy/ventoy_4k.disk.img.xz` — Legacy BIOS 137 GiB Limitation Workaround: Copy these files **first** to your ISO partition — they must reside within the first < 128 GiB of the disk to function properly.

3. Finalize, boot, and enjoy!
   
   - Safely eject the USB from your host OS.
   - Insert into any x86_64 machine and boot from USB.

## Ventoy Setup & Customization

- **Menu Aliases & Tips**  
  - Aliases ensure consistent naming across different folder depths.  
  - Custom tips display beneath each entry using a **patched TrueType font** to match the theme.

- **Maximized gfxmode**
  - Setting "gfxmode": "max" forces the highest supported framebuffer resolution on each machine, ensuring the theme’s fine details and icons are always fully visible without distortion.

- **Persistence**
  - To enable persistence in selected live distributions, the configuration was defined in the `ventoy.json`. This allows system changes, installed packages, and user files to be retained across reboots. Persistence files are kept under the `/Persistence` directory and obtained from [Ventoy backend releases](https://github.com/ventoy/backend/releases). See [Ventoy persistence plugin](https://www.ventoy.net/en/plugin_persistence.html).

## Theme Customization
- Based on [CRT‑Amber‑GRUB‑Theme](https://github.com/Jacksaur/CRT-Amber-GRUB-Theme).  
- Personalized icons for each category (desktop, server, NAS, security, utilities, Windows).  
- Patched Ventoy’s GRUB default font so that **menu tips** render in the same pixel‑perfect style as the theme.

### Redrawn Icons
Every category icon (`Desktop`, `Enterprise`, `Handheld`, `Security`, `Storage`, `Utilities`, `Windows`) has been redrawn at 64×64 px to match the theme’s amber phosphor glow and ensure crisp rendering at any resolution.

### Patched Font
The `VTOYEFI/grub/fonts/unicode.pf2` font was patched to include the theme's `fixedsys-regular-32.pf2` used by custom menu tips, guaranteeing that tips render seamlessly in the same bitmap style as the rest of the menu.
